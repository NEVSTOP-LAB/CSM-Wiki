## ⚙️ CSM-INI-Static-Variable-Support（INI/静态变量插件）

> PR: [Dev_2026Q1 #94](https://github.com/NEVSTOP-LAB/CSM-INI-Static-Variable-Support/pull/94)（已合并，2026-03-24）
> PR: [docs: sync and polish README #96](https://github.com/NEVSTOP-LAB/CSM-INI-Static-Variable-Support/pull/96)（已合并，2026-03-28）

### 新增 API：从字符串加载配置变量

关联 Issue：[#95](https://github.com/NEVSTOP-LAB/CSM-INI-Static-Variable-Support/issues/95)

本周期最重要的功能新增——**`CSM - Load Configuration Variables From String.vi`**。

此前，INI 变量只能从文件路径加载配置；新 API 支持直接传入配置字符串内容，非常适合以下场景：

- 配置嵌入在代码或资源中，不需要落盘
- 动态生成配置内容后直接注入
- 测试时不依赖外部 INI 文件

文档中同步补充了"从字符串加载配置变量"的使用示例与说明。

### Bug 修复：INI-Viewer 窗口未打开

修复了从 Tool Launcher 点击后 **INI-Viewer 窗口未能正常打开**的问题，改善了开发调试体验。

### 文档同步与工具图标

- 同步并润色了中英文 README，修复两种语言内容漂移，提升可读性（PR [#96](https://github.com/NEVSTOP-LAB/CSM-INI-Static-Variable-Support/pull/96)）
- 更新了工具图标
- 更新了 `Run Testcases.yml` CI 配置

## 🌐 CSM-TCP-Router-App（TCP 路由应用）

> PR: [Dev_2026Q1 #27](https://github.com/NEVSTOP-LAB/CSM-TCP-Router-App/pull/27)（已合并，2026-03-28）

本次 TCP Router 的更新是本周期变更最为密集的部分，涵盖新功能、协议文档和多个 Bug 修复。

### 新功能：绑定到指定网卡

关联 Issue：[#23](https://github.com/NEVSTOP-LAB/CSM-TCP-Router-App/issues/23)

为 bind server 导出 `Net Address` 参数，支持将服务绑定到指定网卡或网络适配器。这在多网卡环境（如同时连接内网和外网）下非常实用，可精确控制服务监听范围。

### 协议文档完善

关联 Issue：[#24](https://github.com/NEVSTOP-LAB/CSM-TCP-Router-App/issues/24)

- 新增"**指令响应（cmd-resp）**"与"**订阅中断广播返回**"相关数据包类型定义
- 补充 `cmd-resp` 握手说明与通信流程图
- 统一 error 包描述
- 补充 info packet 说明（连接/断开等生命周期事件）
- 同步并完善中英文协议文档与 README

### Bug 修复：多客户端并发接收

关联 Issue：[#25](https://github.com/NEVSTOP-LAB/CSM-TCP-Router-App/issues/25)

修复了多 client 并发时**无法同时接收消息**的问题。此前在并发场景下，消息可能只被其中一个 client 收到，现已修复。

### Bug 修复 + 增强：Interrupt 模式

关联 Issue：[#26](https://github.com/NEVSTOP-LAB/CSM-TCP-Router-App/issues/26)

修复并增强了 `>>` 在 interrupt 场景的行为，完成 interrupt mode 完整实现，并新增 **Example 6** 进行演示。

### 仓库清理

删除 `CSM-TCP-Router.lvcsm` 文件并加入 `.gitignore`，避免自动生成文件被意外提交。
