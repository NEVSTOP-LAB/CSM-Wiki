# v2025.January: 安装界面优化与工具改进

> 【v2025.January 精简公告】CSM v2025.January 已发布并推送至 VIPM。本期亮点：全新安装对话框展示 CSM 简介、新增 `.lvcsm` 配置文件统一管理工具设置、API String 增强 Cluster 子元素解析能力，以及新增 TCP 应用示例。

2025 年 1 月的 CSM 更新已经推送到 VIPM。Communicable State Machine Framework - Package List. 如果你觉得 CSM 对你有帮助，请在 GitHub/Gitee 上 star 对应的仓库！也欢迎有经验的 LabVIEW 高手参与到 CSM 的开发中来。

- GitHub: [https://github.com/NEVSTOP-LAB](https://github.com/NEVSTOP-LAB)
- gitee: [https://gitee.com/NEVSTOP-LAB](https://gitee.com/NEVSTOP-LAB)
- vipm: [Communicable State Machine Framework - Package List](https://www.vipm.io/package/nevstop_lib_communicable_state_machine/)
- VIPM 打包好的 Release Package(VIPC) 合集（包含全部 Addon + Examples，可以离线安装）: [https://pan.baidu.com/s/10fsnFmJpn-P_HLbpH9IFLg](https://pan.baidu.com/s/10fsnFmJpn-P_HLbpH9IFLg) 提取码: CSMF

## 版本亮点

- 安装对话框全面升级（[#342](https://github.com/NEVSTOP-LAB/Communicable-State-Machine/issues/342)），新界面展示 CSM 简介，提升初次安装体验。
- 新增 `.lvcsm` 配置文件（[#368](https://github.com/NEVSTOP-LAB/Communicable-State-Machine/issues/368)），统一存储 CSM 工具的全部配置，方便团队共享设置。
- CSM Tool Launcher 加载速度显著提升（[#353](https://github.com/NEVSTOP-LAB/Communicable-State-Machine/issues/353)），并集成 Debug 工具帮助说明。
- API String 增强 Cluster 子元素解析（[#71](https://github.com/NEVSTOP-LAB/CSM-API-String-Arguments-Support/issues/71)、[#76](https://github.com/NEVSTOP-LAB/CSM-API-String-Arguments-Support/issues/76)），支持仅解析部分元素及 `<sub-cluster>.<leaf>` 形式的 tag 名。
- TCP Router 新增 TCP Server/Client 应用示例，展示如何结合 CSM 与 JKI TCP Server 搭建网络通信。

## 按仓库变更

### 1. <u>Communicable-State-Machine</u>

#### 功能改动（重点）

**模板（Template）**

- [update] [#359](https://github.com/NEVSTOP-LAB/Communicable-State-Machine/issues/359) 优化模板 While Loop 尺寸。感谢 @casey502。

**全局日志（GlobalLog）**

- [fix] [#372](https://github.com/NEVSTOP-LAB/Communicable-State-Machine/issues/372)、[#371](https://github.com/NEVSTOP-LAB/Communicable-State-Machine/issues/371) 修复注册与注销动作的日志精度，显示更准确的操作记录。

**工具（Tool）**

- [add] [#368](https://github.com/NEVSTOP-LAB/Communicable-State-Machine/issues/368) 新增 `.lvcsm` 文件，用于存储 CSM 工具的全部配置。
- [update] 高亮工具不再对参数使用下划线标注。
- [update] [#353](https://github.com/NEVSTOP-LAB/Communicable-State-Machine/issues/353) 提升 CSM Tool Launcher 加载速度。
- [update] 在 CSM Tool Launcher 中展示 CSM Debug 工具的帮助内容。
- [update] 在 CSM Debug Console 的状态列表中新增"仅 API"（API Only）选项。
- [fix] [#327](https://github.com/NEVSTOP-LAB/Communicable-State-Machine/issues/327) 修复 CSM Tool Launcher 应作为单例运行的问题。

**CSMHelper**

- [add] [#370](https://github.com/NEVSTOP-LAB/Communicable-State-Machine/issues/370) 新增 `Jump to CSM State.vi`，供 CSM 工具与插件调用。

**安装程序（Installer）**

- [update] [#342](https://github.com/NEVSTOP-LAB/Communicable-State-Machine/issues/342) 升级安装对话框界面，展示 CSM 简介。感谢 @KivenJia。

#### 文档与图标（简述）

- [update] 小幅修复/更新示例（[#355](https://github.com/NEVSTOP-LAB/Communicable-State-Machine/issues/355)、[#357](https://github.com/NEVSTOP-LAB/Communicable-State-Machine/issues/357)）。
- [fix] [#365](https://github.com/NEVSTOP-LAB/Communicable-State-Machine/issues/365) 修复 `0. Base Concepts/CSM Data Type.vi` 中的数据名称错误。感谢 @MapleBay。

### 2. <u>CSM-API-String-Arguments-Support</u>

#### 功能改动（重点）

- [important] [#71](https://github.com/NEVSTOP-LAB/CSM-API-String-Arguments-Support/issues/71) 为 Cluster 数据类型新增选项，支持仅解析字符串中的部分元素。
- [important] [#76](https://github.com/NEVSTOP-LAB/CSM-API-String-Arguments-Support/issues/76) `<sub-cluster>.<leaf-element name>` 格式现可作为 Cluster 数据类型的 tag 名。
- [update] [#77](https://github.com/NEVSTOP-LAB/CSM-API-String-Arguments-Support/issues/77) `Convert Data To API String Arguments.vim` 支持无 tag 模式（No-tag mode）。
- [update] [#74](https://github.com/NEVSTOP-LAB/CSM-API-String-Arguments-Support/issues/74) 跟随 CSM Core 更新，简化相关模板。
- [update] [#73](https://github.com/NEVSTOP-LAB/CSM-API-String-Arguments-Support/issues/73) 更新 `CSM - Convert HexStr to Data.vim` 的连接器面板。
- [update] 改善空字符串场景的处理逻辑。
- [fix] [#75](https://github.com/NEVSTOP-LAB/CSM-API-String-Arguments-Support/issues/75) `Set Float Format String.vi` 现已支持 `"%_6g"` 等格式字符串。

### 3. <u>CSM-INI-Static-Variable-Support</u>

#### 功能改动（重点）

- [fix] 修复在某些边界情况下默认 INI 配置路径错误的问题。

### 4. <u>CSM-TCP-Router-App</u>

#### 功能改动（重点）

- [add] 新增 TCP Server/Client 应用示例，演示如何利用 CSM 框架与 JKI TCP Server 搭建 TCP 网络通信。

### 5. <u>CSM-Continuous-Meausrement-and-Logging</u>

#### 文档与图标（简述）

- [update] 更新包图标。

### 6. <u>其他仓库</u>

- CSM-MassData-Parameter-Support：本期无 `v2025.January` 发布，已忽略。

## 按功能变更

### 1. <u>功能新增</u>

- 新增 `.lvcsm` 配置文件机制，统一管理 CSM 工具配置（Core [#368](https://github.com/NEVSTOP-LAB/Communicable-State-Machine/issues/368)）。
- 新增 `Jump to CSM State.vi` 辅助 VI，供工具和插件复用（Core [#370](https://github.com/NEVSTOP-LAB/Communicable-State-Machine/issues/370)）。
- API String 支持 Cluster 部分元素解析与 No-tag 模式（[#71](https://github.com/NEVSTOP-LAB/CSM-API-String-Arguments-Support/issues/71)、[#77](https://github.com/NEVSTOP-LAB/CSM-API-String-Arguments-Support/issues/77)）。
- TCP Router 新增 TCP Server/Client 完整应用示例。
- Debug Console 新增"仅 API"状态列表过滤选项。

### 2. <u>改进优化</u>

- CSM Tool Launcher 加载速度优化（[#353](https://github.com/NEVSTOP-LAB/Communicable-State-Machine/issues/353)），启动更流畅。
- 安装界面全新改版，展示 CSM 简介（[#342](https://github.com/NEVSTOP-LAB/Communicable-State-Machine/issues/342)）。
- API String Cluster 支持 `<sub-cluster>.<leaf>` tag 格式（[#76](https://github.com/NEVSTOP-LAB/CSM-API-String-Arguments-Support/issues/76)）。
- 高亮工具视觉优化，不再对参数使用下划线。
- CSM Tool Launcher 集成 Debug 工具帮助说明，减少查阅文档的需要。

### 3. <u>问题修复</u>

- 修复 CSM Tool Launcher 未作为单例运行的问题（[#327](https://github.com/NEVSTOP-LAB/Communicable-State-Machine/issues/327)）。
- 修复注册与注销日志精度问题（[#372](https://github.com/NEVSTOP-LAB/Communicable-State-Machine/issues/372)、[#371](https://github.com/NEVSTOP-LAB/Communicable-State-Machine/issues/371)）。
- 修复示例中的数据名称错误（[#365](https://github.com/NEVSTOP-LAB/Communicable-State-Machine/issues/365)）。
- 修复 INI 插件默认配置路径边界情况错误。
- 修复 API String `Set Float Format String.vi` 对特定格式字符串的支持（[#75](https://github.com/NEVSTOP-LAB/CSM-API-String-Arguments-Support/issues/75)）。

## 破坏性变更

本期无破坏性变更。

## 升级指引

本期升级无特殊操作，直接通过 VIPM 更新即可。

如果项目使用了 API String 的 Cluster 类型解析，建议验证 No-tag 模式与 `<sub-cluster>.<leaf>` tag 名的新行为是否与现有逻辑兼容。
