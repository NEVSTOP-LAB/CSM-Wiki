# v2025.September: lvlibp支持

> 【v2025.September 精简公告】CSM v2025.September 已发布并推送至 VIPM。本期最重要特性：核心框架与全部主要插件（API String、INI、MassData、TCP Router）同步支持将 CSM 模块打包为 lvlibp（Packed Library），解锁 CSM 在项目封装与发布中的新能力。

2025 年 9 月的 CSM 更新已经推送到 VIPM。Communicable State Machine Framework - Package List. 如果你觉得 CSM 对你有帮助，请在 GitHub/Gitee 上 star 对应的仓库！也欢迎有经验的 LabVIEW 高手参与到 CSM 的开发中来。

- GitHub: [https://github.com/NEVSTOP-LAB](https://github.com/NEVSTOP-LAB)
- gitee: [https://gitee.com/NEVSTOP-LAB](https://gitee.com/NEVSTOP-LAB)
- vipm: [Communicable State Machine Framework - Package List](https://www.vipm.io/package/nevstop_lib_communicable_state_machine/)
- VIPM 打包好的 Release Package(VIPC) 合集（包含全部 Addon + Examples，可以离线安装）: [https://pan.baidu.com/s/10fsnFmJpn-P_HLbpH9IFLg](https://pan.baidu.com/s/10fsnFmJpn-P_HLbpH9IFLg) 提取码: CSMF

## 版本亮点

- **全系 lvlibp 支持**：Core（[#502](https://github.com/NEVSTOP-LAB/Communicable-State-Machine/issues/502)）、API String（[#95](https://github.com/NEVSTOP-LAB/CSM-API-String-Arguments-Support/issues/95)）、INI、MassData（[#26](https://github.com/NEVSTOP-LAB/CSM-MassData-Parameter-Support/issues/26)）、TCP Router 同步支持 CSM 模块打包为 lvlibp，让 CSM 模块可以作为 Packed Library 发布和复用。
- **修复 lvlibp 异步运行崩溃**（[#499](https://github.com/NEVSTOP-LAB/Communicable-State-Machine/issues/499)）：当 CSM 模块被打包为 lvlibp 且以异步方式运行时，此前存在的关键错误已被修复。
- INI 插件提升 `Sync to File.vi` 的性能（[#87](https://github.com/NEVSTOP-LAB/CSM-INI-Static-Variable-Support/issues/87)）。
- 全局变量使用方式优化（[#504](https://github.com/NEVSTOP-LAB/Communicable-State-Machine/issues/504)），改用更可靠的机制替代全局变量作为变更标志。

## 按仓库变更

### 1. <u>Communicable-State-Machine</u>

#### 功能改动（重点）

- [feature] [#502](https://github.com/NEVSTOP-LAB/Communicable-State-Machine/issues/502) 支持 CSM 模块打包为 lvlibp。
- [fix] [#499](https://github.com/NEVSTOP-LAB/Communicable-State-Machine/issues/499) 修复 CSM 模块打包为 lvlibp 后异步运行时的关键错误（Critical Error）。
- [update] [#504](https://github.com/NEVSTOP-LAB/Communicable-State-Machine/issues/504) 不再使用全局变量作为变更标志。
- [update] [#501](https://github.com/NEVSTOP-LAB/Communicable-State-Machine/issues/501) 关键错误改用 ErrString 表示。
- [update] 改进调板与文档说明。

### 2. <u>CSM-API-String-Arguments-Support</u>

#### 功能改动（重点）

- [feature] [#95](https://github.com/NEVSTOP-LAB/CSM-API-String-Arguments-Support/issues/95) 支持 lvlibp。

### 3. <u>CSM-INI-Static-Variable-Support</u>

#### 功能改动（重点）

- [feature] 支持 lvlibp 特性。
- [fix] [#87](https://github.com/NEVSTOP-LAB/CSM-INI-Static-Variable-Support/issues/87) 提升 `Sync to File.vi` 的性能。

### 4. <u>CSM-MassData-Parameter-Support</u>

#### 功能改动（重点）

- [feature] [#26](https://github.com/NEVSTOP-LAB/CSM-MassData-Parameter-Support/issues/26) MassData 现在可以与 lvlibp 一起使用。
- [update] 文档更新。

### 5. <u>CSM-TCP-Router-App</u>

#### 功能改动（重点）

- 本期同步 lvlibp 支持，与核心版本对齐。

### 6. <u>其他仓库</u>

- CSM-Continuous-Meausrement-and-Logging：本期无 `v2025.September` 发布，已忽略。

## 按功能变更

### 1. <u>功能新增</u>

- 全系支持 lvlibp（Packed Library）打包模式，覆盖 Core、API String、INI、MassData、TCP Router 五个仓库。

### 2. <u>改进优化</u>

- 全局变量使用方式优化，提升稳定性（[#504](https://github.com/NEVSTOP-LAB/Communicable-State-Machine/issues/504)）。
- 关键错误改用 ErrString 表示，便于捕获和传递（[#501](https://github.com/NEVSTOP-LAB/Communicable-State-Machine/issues/501)）。
- INI `Sync to File.vi` 性能提升（[#87](https://github.com/NEVSTOP-LAB/CSM-INI-Static-Variable-Support/issues/87)）。
- 调板与文档持续完善。

### 3. <u>问题修复</u>

- 修复 CSM 模块打包为 lvlibp 后异步运行崩溃的关键错误（[#499](https://github.com/NEVSTOP-LAB/Communicable-State-Machine/issues/499)）。

## 破坏性变更

本期无破坏性变更。

## 升级指引

1. 直接通过 VIPM 更新即可完成本次升级。
2. 如果项目计划将 CSM 模块打包为 lvlibp，升级到本版本后即可开始尝试，同时建议在测试环境中充分验证异步启动和资源管理逻辑。
3. 本期同步了 lvlibp 支持到所有主要插件，打包前请确认所有用到的插件均已更新至 `v2025.September`。
