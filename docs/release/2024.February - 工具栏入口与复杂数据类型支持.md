# v2024.Feb: 工具栏入口与复杂数据类型支持

> 【v2024.Feb 精简公告】CSM v2024.Feb 已发布并推送至 VIPM。本期核心亮点是在 LabVIEW 工具栏新增了 CSM 工具启动入口，显著降低了工具访问成本；同时新增了消息过滤 VI，并修复了 API String 插件中复数类型的解析 bug。

2024 年 2 月（实际发布于 3 月初）的 CSM v2024.Feb 更新已经推送到 VIPM。如果你觉得 CSM 对你有帮助，请在 GitHub/Gitee 上 star 对应的仓库！也欢迎有经验的 LabVIEW 高手参与到 CSM 的开发中来。

- VIPM: [Communicable State Machine Framework - Package List](https://www.vipm.io/package/nevstop_lib_communicable_state_machine/)
- VIPM 打包好的 Release Package(VIPC) 合集（包含全部 Addon + Examples，可以离线安装）: [https://pan.baidu.com/s/10fsnFmJpn-P_HLbpH9IFLg](https://pan.baidu.com/s/10fsnFmJpn-P_HLbpH9IFLg) 提取码: CSMF

## 版本亮点

- **工具栏快捷入口**（[#252](https://github.com/NEVSTOP-LAB/Communicable-State-Machine/issues/252)）：LabVIEW 工具栏新增 CSM 工具启动按钮，一键打开 Debug Console、Log Viewer 等所有 CSM 工具，极大提升了开发效率。
- **新增消息过滤 VI**（[#251](https://github.com/NEVSTOP-LAB/Communicable-State-Machine/issues/251)）：`CSM - Filter Messages to Non-Existing Modules.vi` 可过滤掉发送给不存在模块的消息，避免无效消息堆积。
- **修复复数类型 bug**（[#28](https://github.com/NEVSTOP-LAB/CSM-API-String-Arguments-Support/issues/28)）：API String 插件中复数数据类型的解析错误已被修复，使用复数参数的应用可以正常工作。
- **用户体验改进**（[#245](https://github.com/NEVSTOP-LAB/Communicable-State-Machine/issues/245)）：多项 UI 和交互细节改进，让开发过程更加顺畅。
- **图标与面板更新**：示例 VI 和面板图标全面刷新，视觉效果更加统一。

## 按仓库变更

### 1. <u>Communicable-State-Machine</u>

#### 功能改动（重点）

- **工具栏入口**（[#252](https://github.com/NEVSTOP-LAB/Communicable-State-Machine/issues/252)）：在 LabVIEW 工具栏新增 CSM 工具启动入口，开发者无需手动导航菜单即可快速启动所有 CSM 工具。
- **新增消息过滤 VI**（[#251](https://github.com/NEVSTOP-LAB/Communicable-State-Machine/issues/251)）：`CSM - Filter Messages to Non-Existing Modules.vi` 允许在发送消息前过滤掉目标模块不存在的消息，减少不必要的错误处理逻辑。
- **改进后台线程退出逻辑**：使用 watchdog 机制确保后台线程安全退出，提升框架稳定性。
- **用户体验改进**（[#245](https://github.com/NEVSTOP-LAB/Communicable-State-Machine/issues/245)）：根据用户反馈改进了多处交互细节。
- **次要修复**（[#254](https://github.com/NEVSTOP-LAB/Communicable-State-Machine/issues/254)）：修复了一处已知的小问题。
- **修复 VIPM.io 链接**（[#246](https://github.com/NEVSTOP-LAB/Communicable-State-Machine/issues/246)）：修复了 VI 描述中指向 VIPM.io 的失效链接。

#### 文档与图标（简述）

- 更新了示例 VI 图标和调色板图标，统一视觉风格。

### 2. <u>CSM-API-String-Arguments-Support</u>

#### 功能改动（重点）

- **修复复数数据类型 bug**（[#28](https://github.com/NEVSTOP-LAB/CSM-API-String-Arguments-Support/issues/28)）：修复了上一版本（v2024.Jan）引入的复数类型解析错误，SGLComplex 和 DBLComplex 类型现在可以正确编解码。

#### 文档与图标（简述）

- 与核心库版本同步更新。

### 3. <u>其他仓库</u>

- **CSM-MassData-Parameter-Support**：本期无该版本发布，已忽略。
- **CSM-TCP-Router**：本期无该版本发布，已忽略。

## 按功能变更

### 1. <u>功能新增</u>

- 新增 `CSM - Filter Messages to Non-Existing Modules.vi`，在消息发出前自动过滤目标不存在的消息（[#251](https://github.com/NEVSTOP-LAB/Communicable-State-Machine/issues/251)）。
- LabVIEW 工具栏新增 CSM 工具启动入口按钮（[#252](https://github.com/NEVSTOP-LAB/Communicable-State-Machine/issues/252)）。

### 2. <u>改进优化</u>

- 示例 VI 图标与调色板图标全面更新。

### 3. <u>问题修复</u>

- 修复 API String 插件中复数数据类型的解析 bug（[#28](https://github.com/NEVSTOP-LAB/CSM-API-String-Arguments-Support/issues/28)）。
- 修复 VI 描述中的 VIPM.io 链接失效问题（[#246](https://github.com/NEVSTOP-LAB/Communicable-State-Machine/issues/246)）。
- 次要修复（[#254](https://github.com/NEVSTOP-LAB/Communicable-State-Machine/issues/254)）。

## 破坏性变更

本期无破坏性变更。

## 升级指引

1. 通过 VIPM 更新到 v2024.Feb 版本。
2. 如果之前因为 API String 复数类型 bug 绕过了问题，升级后可以移除相关 workaround 代码。
3. 新的工具栏入口为可选使用，不影响现有代码。
