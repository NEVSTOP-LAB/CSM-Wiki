# 2026Q1: 核心优化与插件增强

> 【2026Q1 精简公告】CSM 2026Q1 已发布并推送至 VIPM。本期重点包括：核心框架性能与日志可观测性优化、API String 解析修复、INI 插件新增字符串加载配置 API、TCP Router 协议与网络绑定能力增强。建议从 v2025.Q4 升级并重点回归脚本解析、链式退出与网络协议流程。

## 版本亮点

- 核心框架在共享 VI 内联、日志完整性与链式模式退出稳定性上完成一轮集中优化，重点改善性能、可观测性与异常退出行为。
- API String 参数支持增强了字符串安全策略与数字后缀解析，减少文本参数与数值参数互转时的歧义。
- INI 静态变量插件新增字符串加载配置 API，便于在脚本化部署和远程配置场景中快速注入参数。
- TCP Router 应用新增绑定网卡地址导出、协议包类型补充与文档完善，提升网络部署和联调效率。

## 按仓库变更

### 1. _Communicable-State-Machine_

#### 功能改动（重点）

- [update] [#587](https://github.com/NEVSTOP-LAB/Communicable-State-Machine/issues/587) 将 `_shared/` 目录中的 VIs 转为 inline，用于性能优化。
- [fix] [#591](https://github.com/NEVSTOP-LAB/Communicable-State-Machine/issues/591) 修复 chain 模式在 `macro: exit` 且节点超时时的退出问题，降低链式流程残留风险。
- [update] [#586](https://github.com/NEVSTOP-LAB/Communicable-State-Machine/issues/586)、[#590](https://github.com/NEVSTOP-LAB/Communicable-State-Machine/issues/590) 完善 Sync/Async 消息 complete log，并在 worker/chain 模式销毁日志中增加剩余节点数量，便于诊断尾部清理行为。
- [add/fix/remove] 调试工具侧新增通用 UI 工具 VI，修复 Interface Browser 启动入口，并移除 VI Analyzer 相关支持（[#326](https://github.com/NEVSTOP-LAB/Communicable-State-Machine/issues/326)、[#321](https://github.com/NEVSTOP-LAB/Communicable-State-Machine/issues/321)）。
- [doc] [#602](https://github.com/NEVSTOP-LAB/Communicable-State-Machine/issues/602) 将 Syntax 文档重写为中英双语语法参考，降低学习门槛。

#### 用户收益

- 在高频消息模块中，内联优化通常可降低调用开销。
- 异常退出和模块销毁的观测信息更完整，定位问题更快。
- 调试工具更聚焦日常开发路径，减少历史工具带来的维护负担。

### 2. _CSM-MassData-Parameter-Support_

#### 功能改动（重点）

- 文档更新 [#33](https://github.com/NEVSTOP-LAB/CSM-MassData-Parameter-Support/issues/33)：MassData 参数默认缓存大小描述由 `1MB` 更新为 `50MB`（中英文同步）。

#### 文档与图标（简述）

- 更新工具图标。

### 3. _CSM-API-String-Arguments-Support_

#### 功能改动（重点）

- [update] [#109](https://github.com/NEVSTOP-LAB/CSM-API-String-Arguments-Support/issues/109) 对 string datatype 禁用 `<safestr>`，减少字符串类型的语义歧义。
- [fix] [#105](https://github.com/NEVSTOP-LAB/CSM-API-String-Arguments-Support/issues/105) 修复带 `B`（byte）后缀数值（如 `100B`）的解析与格式化问题。
- [fix] [#107](https://github.com/NEVSTOP-LAB/CSM-API-String-Arguments-Support/issues/107) 修复 `Strip Trailing Zeros for Float String.vi` 潜在缺陷，提升浮点字符串输出稳定性。

### 4. _CSM-INI-Static-Variable-Support_

#### 功能改动（重点）

- [add] [#95](https://github.com/NEVSTOP-LAB/CSM-INI-Static-Variable-Support/issues/95) 新增 API：`CSM - Load Configuration Variables From String.vi`。
- [fix] [#94](https://github.com/NEVSTOP-LAB/CSM-INI-Static-Variable-Support/issues/94) 修复工具启动器点击后 INI-Viewer 窗口未打开的问题。

#### 文档与图标（简述）

- 更新工具图标。

### 5. _CSM-TCP-Router-App_

#### 功能改动（重点）

- [PR #27](https://github.com/NEVSTOP-LAB/CSM-TCP-Router-App/pull/27) 合并 2026Q1 版本。
- [#23](https://github.com/NEVSTOP-LAB/CSM-TCP-Router-App/issues/23)：新增 bind server 的 `Net Address` 导出能力，可显式绑定到指定网卡/网络适配器。
- 协议包类型更新：新增“指令响应”“订阅中断广播返回”等相关数据包定义。
- [#24](https://github.com/NEVSTOP-LAB/CSM-TCP-Router-App/issues/24)：同步增强中英文协议文档与 README，补充 `cmd-resp` 握手、通信流程图、error 包说明与 info packet 生命周期说明。

## 按功能变更

### 1. _功能新增_

- INI 插件新增 `CSM - Load Configuration Variables From String.vi`，支持通过字符串快速装载配置。
- TCP Router 扩展网络绑定与协议事件类型，提升路由层可配置性。

### 2. _改进优化_

- 核心框架共享 VI 内联优化。
- 全局日志与模块销毁日志观测维度增强。
- API String 参数处理对 byte 后缀与浮点尾零处理更稳健。

### 3. _问题修复_

- chain 模式在超时+退出场景下的退出缺陷。
- INI-Viewer 启动入口无响应问题。
- API String 数值后缀和浮点格式处理问题。

## 破坏性变更

- 本期未发现明确的破坏性 API 变更。
- 注意：API String 中针对 string datatype 的 `<safestr>` 策略调整，若你的脚本依赖历史行为，建议在升级前验证相关解析路径。

## 升级指引

1. 优先从 `v2025.Q4` 升级到 `v2026.Q1`，并在测试环境回归脚本解析、链式退出与日志采集流程。
2. 使用 API String 的项目，重点检查带 `B` 后缀参数和字符串参数的兼容性。
3. 使用 TCP Router 的项目，建议按新文档核对协议包类型和 `cmd-resp` 握手流程。
4. 需要动态注入配置的项目，可评估接入 INI 新增字符串加载 API 以简化部署。
