# v2024.Oct: 状态事件修复与浮点格式API

> 【v2024.Oct 精简公告】CSM v2024.October 已发布并推送至 VIPM。本期修复了一个关键 bug：CSM 状态变更事件不工作的问题（[#328](https://github.com/NEVSTOP-LAB/Communicable-State-Machine/issues/328)）；同时新增了 VI 说明自动本地化功能（[#330](https://github.com/NEVSTOP-LAB/Communicable-State-Machine/issues/330)）、浮点数格式字符串设置 API（[#66](https://github.com/NEVSTOP-LAB/CSM-API-String-Arguments-Support/issues/66)）和 INI 插件重置到默认值功能。

2024 年 10 月（实际发布于 11 月初）的 CSM v2024.October 更新已经推送到 VIPM。如果你觉得 CSM 对你有帮助，请在 GitHub/Gitee 上 star 对应的仓库！也欢迎有经验的 LabVIEW 高手参与到 CSM 的开发中来。

## 版本亮点

- **状态变更事件 bug 修复**（[#328](https://github.com/NEVSTOP-LAB/Communicable-State-Machine/issues/328)）：修复了 CSM Status Change Event 在特定条件下不触发的严重 bug，确保依赖状态事件的应用能正常工作。
- **VI 说明自动本地化**（[#330](https://github.com/NEVSTOP-LAB/Communicable-State-Machine/issues/330)）：安装后会自动本地化 VI 说明，用户无需手动执行语言切换工具即可获得本地化的 VI 描述。
- **浮点格式字符串设置 API**（[#66](https://github.com/NEVSTOP-LAB/CSM-API-String-Arguments-Support/issues/66)）：API String 插件新增 `Set Float Format String.vi`，允许开发者通过完整的 LabVIEW 格式字符串精确控制浮点数的输出格式。
- **INI 插件重置到默认值**：新增 `Reset to Default.vi` API，支持一键将配置变量重置回默认值。
- **命令调试工具增强**：Updated Command Debug Tool 在切换 CSM 模块时会自动列出可用命令，提升调试效率。

## 按仓库变更

### 1. <u>Communicable-State-Machine</u>

#### 功能改动（重点）

- **修复 CSM Status Change Event**（[#328](https://github.com/NEVSTOP-LAB/Communicable-State-Machine/issues/328)）：修复了关键 bug，状态变更事件在之前版本中存在不触发的问题，本次修复确保事件能够可靠触发。
- **VI 说明自动本地化**（[#330](https://github.com/NEVSTOP-LAB/Communicable-State-Machine/issues/330)）：安装完成后，框架会自动检测 LabVIEW 语言设置并本地化 VI 说明，无需用户手动运行语言切换工具。
- **更新 Switch Language 工具**：改进了语言切换工具的功能和用户体验。
- **更新命令调试工具**：Command Debug Tool 在切换监控的 CSM 模块时，现在会自动列出该模块支持的所有命令，无需手动查阅文档。
- **次要修复**（[#325](https://github.com/NEVSTOP-LAB/Communicable-State-Machine/issues/325)、[#326](https://github.com/NEVSTOP-LAB/Communicable-State-Machine/issues/326)）：修复了两处已报告的小问题。

#### 文档与图标（简述）

- 更新了 zh-cn 版本的 VI 说明 Markdown 帮助文件。
- 与核心库版本同步更新。

### 2. <u>CSM-API-String-Arguments-Support</u>

#### 功能改动（重点）

- **浮点格式字符串设置 API**（[#66](https://github.com/NEVSTOP-LAB/CSM-API-String-Arguments-Support/issues/66)）：新增 `Set Float Format String.vi`，开发者可以传入 LabVIEW 标准格式字符串（如 `%+.4f`、`%e` 等）来精确控制浮点数在 API String 中的输出格式，满足各类精度和格式化需求。

#### 文档与图标（简述）

- 与核心库版本同步更新。

### 3. <u>CSM-INI-Static-Variable-Support</u>

#### 功能改动（重点）

- **新增 `Reset to Default.vi` API**：允许在运行时将配置变量重置到初始默认值，简化了需要恢复出厂配置的场景。
- **面板子面板重组**：更新了面板布局，通过重新组织和添加子面板使 API 分组更加清晰。
- **增强配置文件带后缀加载支持**：改进了对带有文件名后缀的配置文件的加载支持。

#### 文档与图标（简述）

- 与核心库版本同步更新。

### 4. <u>其他仓库</u>

- **CSM-MassData-Parameter-Support**：本期无该版本发布，已忽略。
- **CSM-TCP-Router**：本期无该版本发布，已忽略。

## 升级指引

1. 通过 VIPM 更新到 v2024.October 版本。
2. 若你的应用依赖 CSM Status Change Event，强烈建议尽快升级，以确保事件能正常触发（[#328](https://github.com/NEVSTOP-LAB/Communicable-State-Machine/issues/328)）。
3. 升级后 VI 说明会自动本地化，无需手动运行语言切换工具。
4. 若需要精确控制浮点数格式，可以使用新的 `Set Float Format String.vi`（[#66](https://github.com/NEVSTOP-LAB/CSM-API-String-Arguments-Support/issues/66)）替代之前的精度设置方式。
